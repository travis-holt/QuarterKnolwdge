Railway Express Server  ·  server.js  ·  auto-deploys on push to main

Serves the SPA from dist/ and proxies all Gemini calls. Never reads from Firestore.

  • POST /api/generate-scenarios AI question drafts for supervisor review
  • POST /api/generate-coaching Personalised post-check coaching notes
  • POST /api/interview-turn Gemini acts as patient caller (roleplay)
  • POST /api/grade-interview AI score + feedback on a practice call
  • POST /api/generate-audit Flawed transcript for "Spot the Error"
  • POST /api/coach-audit AI mentor reply to navigator's reflection
  • GET /api/health Deploy health check
  • GET /* SPA catch-all → index.html
📄 api/_sop-context.js — SOP text (server-side only) Never sent to the client. Two grounding contexts: Pediatrics (from updated SOP PDF) and OB/GYN (sanitized — no real names/numbers). All Gemini calls are grounded in the correct SOP via sopContextFor(dept).
Google Gemini API  ·  model: gemini-2.5-flash

Key stored only in Railway env vars (GEMINI_API_KEYS). Rotated automatically on rate-limit (429/503). All output is advisory — never writes to Firestore directly.

Assessment Data Flow — Navigator Submits a Check
1

Sign in

Navigator picks their name from a dropdown and enters a 4-digit PIN. Roster fetched from Firestore once; PIN validated client-side.

2

Pick department

Navigator chooses Pediatrics or OB/GYN. Active questions for that department are fetched from Firestore (seed fallback if empty).

3

Take the check

Navigator answers ~14–20 scenario MCQs. Answers held in local React state only — nothing is written to the database mid-check.

4

Score & save

On submit: scorePerDomain and scorePerCompetency run in the browser. Result written to Firestore (results collection, composite key).

5

AI coaching

Browser calls POST /api/generate-coaching. Railway sends missed questions + rationales to Gemini. Personalised coaching notes returned and rendered. Nothing written to Firestore.

6

Supervisor sees it live

SupervisorApp has a live onSnapshot subscription. The matrix, floor KPIs, training assignments, and question health dots all update automatically — no page refresh needed.
